# FRODODESK — PROCEDURA UFFICIALE APERTURA CHAT

---

## 🎯 OBIETTIVO

Questa procedura serve per:

- mantenere continuità tra le chat
- evitare perdita di stato
- evitare file enormi inutili
- lavorare in modo modulare e stabile

---

## 📦 FILE DA INVIARE ALL’APERTURA

### 1️⃣ SEMPRE OBBLIGATORI

Inviare sempre:

- `docs/FRODODESK_SYSTEM_STATE.md`
- `docs/FRODODESK_RULES.md`

---

### 2️⃣ MODULO SPECIFICO

Inviare SOLO il modulo su cui si lavora:

Esempi:

- `docs/modules/eventi_alice.md`
- `docs/modules/copertura.md`
- `docs/modules/ui_calendario.md`

⚠️ NON inviare tutti i moduli  
⚠️ solo quello necessario

---

### 3️⃣ FILE CODICE REALE

Inviare SOLO il file reale su cui si lavora:

Esempi:

- `calendario_screen_stepa.dart`
- `coverage_engine.dart`
- altri file specifici

---

## 🧠 MESSAGGIO DI APERTURA

All’inizio della chat scrivere:
FRODODESK — APERTURA

Ti invio:

SYSTEM_STATE
RULES
modulo attivo
file codice reale

Obiettivo di questa chat:
[descrizione chiara del lavoro]


---

## ⚠️ REGOLE OPERATIVE

- lavorare un passo alla volta (CNC)
- non anticipare più step insieme
- non modificare più file insieme
- lavorare sempre su file reali
- chiedere sempre conferma visiva prima di modifiche

---

## 🚫 COSA NON FARE

- non inviare tutti i docs
- non inviare file inutili
- non mescolare più moduli
- non partire senza obiettivo chiaro

---

## 🎯 RISULTATO ATTESO

- chat più leggere
- meno blocchi
- maggiore continuità
- lavoro più preciso
- zero perdita di stato

---
